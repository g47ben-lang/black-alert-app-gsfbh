# 🧪 מדריך גרסת הבדיקה — צבע שחור

גרסת הבדיקה מאפשרת **לשלוח התראות מהמחשב** ולבדוק את כל חוויית ההתראה (צלצול, מסך מלא,
מפה, ניווט) **גם כשאין התראת אמת בשרת**.

> גרסת הבדיקה (`black-alert-v*-test.apk`) מותקנת כאפליקציה **נפרדת** (`com.black.alert.test`,
> בשם "צבע שחור (בדיקה)") **במקביל** לאפליקציה הרגילה — אפשר להחזיק את שתיהן.
> רק גרסת הבדיקה מאפשרת חיבור HTTP לא-מוצפן (לשרת ה-mock). הגרסה הרגילה היא HTTPS בלבד.

---

## מה צריך

1. **גרסת הבדיקה מותקנת** בטלפון (`black-alert-v1.2.0-test.apk` מ-[Releases](../../releases)).
2. **מחשב** עם Python 3 שמריץ את שרת ה-mock (`tools/mock_server.py`).
3. דרך לחבר ביניהם — **USB** (מומלץ) או **אותו Wi-Fi**.

---

## הפעלת שרת ה-mock (במחשב)

```bash
cd tools
python3 mock_server.py            # מאזין על 0.0.0.0:8000
```
פתח בדפדפן **http://localhost:8000/** — תקבל טופס לשליחת התראות בדיקה.

---

## שיטה A — USB (מומלץ, עובד תמיד, גם WSL/טלפון מנוהל)

זו הדרך הנקייה: אין צורך ב-IP, רשת או Firewall.

```bash
# טלפון מחובר ב-USB עם "ניפוי באגים USB" פעיל:
adb reverse tcp:8000 tcp:8000
```
זה גורם ל-`http://127.0.0.1:8000` **בטלפון** להגיע לשרת שעל המחשב.

באפליקציית הבדיקה → **הגדרות → מתקדם — מקור נתונים** → הזן בדיוק:
```
http://127.0.0.1:8000
```
ולחץ **שמור** (מפעיל מחדש את השירות עם המקור החדש).

> ⚠️ הקפד על `http://` בהתחלה ועל הפורט המלא `8000`. כתובת שגויה = לא יקבל התראות.

---

## שיטה B — אותו Wi-Fi (בלי USB)

הרץ את `mock_server.py` על מחשב/שרת שה-IP שלו גלוי ברשת המקומית (לא בתוך WSL — ל-WSL יש
רשת פנימית שהטלפון לא מגיע אליה).

1. מצא את ה-IP של המחשב ברשת (למשל `10.0.0.168`).
2. ודא שה-Firewall מאפשר פורט `8000`.
3. באפליקציה → מקור נתונים → `http://<IP-של-המחשב>:8000`.

---

## שליחת התראת בדיקה

**הכי קל:** בדפדפן `http://localhost:8000/` → מלא את הטופס → **"שלח התראת בדיקה"**.

**או דרך URL** (אפשר גם מהטלפון/סקריפט):
```
http://localhost:8000/push?city=בני ברק&type=3&note=חשוד נעצר&address=רחוב רבי עקיבא 12&lat=32.0874&lng=34.8324&ttl=180
```
| פרמטר | משמעות |
|---|---|
| `city` | שם העיר (כפי שמופיע ב-cities.json) |
| `type` | סוג: `0` מעצר מ.צ · `2` מחסומים · `3` הסגרה · `8` כללי |
| `note` | טקסט חופשי (הערת החמ"ל) |
| `address` | כתובת מדויקת |
| `lat`,`lng` | קואורדינטות לניווט |
| `ttl` | תוקף בשניות (ברירת מחדל 120) |

לסיום אירוע (בדיקת ההתנהגות של `closed`):
```
http://localhost:8000/close?id=<notificationId>
```

תוך ~10 שניות (מסך דולק) האפליקציה תמשוך את האירוע ותתריע.

## איך לבדוק התראה **במסך מלא** (כמו אמת)

מסך מלא קופץ כשהאירוע מגיע **כשהמסך כבוי/נעול** (תרחיש "טלפון בכיס"):

1. ודא שהמקור מוגדר והשירות פעיל.
2. **כבה את מסך הטלפון** (כפתור ההפעלה).
3. שלח התראת בדיקה מהמחשב.
4. תוך שניות — המסך **נדלק לבד**, ההתראה קופצת במסך מלא ומצלצלת.

> אם המסך **דולק והאפליקציה בחזית** כשהאירוע מגיע — תוצג כ-heads-up (באנר עליון) ולא במסך
> מלא. זו התנהגות אנדרואיד תקנית. כבה מסך לפני השליחה כדי לראות את המסך המלא.

---

## מה לבדוק

- ✅ **צלצול + מסך מלא** עם סוג האירוע, עיר, כתובת והערה.
- ✅ **מפה** — מוצגת במכשירים עם WebView. במכשיר כשר/מנוהל ללא WebView מוצגים הקואורדינטות
  במקום (ללא קריסה).
- ✅ **כפתור "נווט"** → פותח **Waze** ישירות למיקום (אם אין Waze — בורר מערכת geo:).
- ✅ **היסטוריה** → לחיצה על אירוע פותחת פרטים מלאים + מפה. מתג "מקומי / שרת".
- ✅ **כפתור "🚨 דיווח על מעצר"** → דיאלוג אישור → חיוג למוקד צבע שחור.
- ✅ **סינון** → הגדרות → בחירת ערים/אזורים, כולל "סמן את כל האזורים".

---

## פתרון תקלות

| בעיה | פתרון |
|---|---|
| לא מקבל התראות | ודא `http://127.0.0.1:8000` מדויק; ש-`adb reverse` פעיל; שהשרת רץ |
| המסך המלא לא קופץ | כבה את מסך הטלפון **לפני** השליחה; אשר הרשאת התראות |
| המפה ריקה | המכשיר ללא WebView/אינטרנט — תקין, יוצגו הקואורדינטות |
| `adb` מציג "more than one device" | הוסף `-s <serial>` (ראה `adb devices`) |

## ניקוי בסיום

```bash
adb reverse --remove-all     # הסרת הגישור
# עצור את mock_server.py (Ctrl+C)
```
