# 📖 צבע שחור — מסמך הסבר מלא

## מה זו המערכת

**"צבע שחור"** (black-alert.com) היא מערכת התראות קהילתית שמתריעה בזמן אמת על אירועים:

| `eventType` | משמעות |
|---|---|
| `0` | נסיון מעצר מ.צ. |
| `2` | התרעת מחסומים |
| `3` | נסיון הסגרה |
| `8` | התראה כללית |

האפליקציה הזו היא לקוח אנדרואיד עצמאי שצורך את **הפיד הציבורי** של המערכת, ומוסיף חוויית
נייד: צלצול, מסך מלא שמדליק את המסך, וניווט בלחיצה.

## ה-API של השרת (FastAPI מאחורי Cloudflare)

### צד קריאה ציבורי (ללא אימות) — מה שהאפליקציה צורכת
| Endpoint | תפקיד |
|---|---|
| `GET /notifications` | אירועים **פעילים** כרגע (מחזיר `[]` כשאין). cache `max-age=5` |
| `GET /alerts-history` | היסטוריית אירועים |
| `GET /lists-versions` | גרסאות רשימות `{"cities":N,"polygons":N}` |
| `GET /static/cities.json` | מיפוי עיר → `{he,en,lat,lng,id,area}` |

### צד כתיבה/חמ"ל (מאומת — HTTPBearer/Google login) — **לא** בשימוש האפליקציה
`POST /report` (פרסום), `/report/update|close|delete` (עריכה/סגירה), `/incoming-alerts`
ו-`/incoming-alerts/{id}/audio` (תור גולמי + אודיו), `/dispatch-lock/*` (תיאום בין דיספאצ'רים),
`/webhook/incoming-alert` (קליטת הדיווח הגולמי).

### מבנה אירוע
```json
{ "notificationId": "evt-…", "cities": ["קרית יערים - טלזסטון"],
  "eventType": 3, "time": 1780607849, "expireAt": 1780613249,
  "version": 1, "status": "closed", "note": "…", "address": "ירושלים",
  "lat": 31.814736, "lng": 35.022204 }
```

## מאיפה מגיעות ההתראות (שרשרת המקור)

1. **מקור גולמי** — קו תוכן ציבורי על **ימות המשיח (call2all)**. אוטומציה מקבלת את השיחה
   הטלפונית ושומרת הקלטה כקובץ, שממתין לתמלול — נשלח ל-`POST /webhook/incoming-alert`
   (טקסט + אודיו).
2. **אישור אנושי בחמ"ל** — דיספאצ'רים מאומתים (Google) מאזינים, ומפרסמים אירוע מאושר דרך
   `POST /report`. הראיות לכך שזה אנושי: מנעול דיספאצ'רים (`dispatch-lock`), חשבונות Google,
   והערות אנושיות-רגשיות בפרסומים.
3. **עריכה/סגירה** — `POST /report/update` מקפיץ `version` (מתריע מחדש), `/report/close`
   מסמן `status:"closed"`.

> **למה ההתראות נערכות:** שלב האישור האנושי הוא בדיוק הסיבה שאותו אירוע מתעדכן/נסגר. לכן
> האפליקציה שומרת **היסטוריה מקומית** שמנציחה כל גרסה — מה שהשרת עלול לדרוס.

## ארכיטקטורת האפליקציה

```
מכשיר תומך (Play Services):   FCM relay (server/) ──push──▶ FcmService ─┐
מכשיר לא-תומך (כשר/מנוהל):     PollingService ──GET /notifications──────┤
                                                                        ▼
                       AlertProcessor  (dedup notificationId:version · סינון · TTL · closed)
                         ├─▶ NotificationHelper → AlertActivity (צלצול + מפה + ניווט Waze)
                         └─▶ HistoryStore (JSONL append-only — שומר עריכות)
```

### עקרונות
- **Polling אמין ברקע** — Foreground Service, מרווח אדפטיבי (מסך דולק מהיר / כבוי איטי) +
  backoff בכשל + watchdog. שורד הריגת מערכת, עולה מחדש אחרי אתחול.
- **Failover ל-push** — מכשירים עם Play Services מקבלים FCM (relay מרכזי); השאר ב-polling.
  גם במצב push יש safety-poll למניעת כשל.
- **חי מול היסטוריה** — צלצול רק על אירוע **פעיל** (`/notifications`), בעוד ההיסטוריה שומרת
  הכל. (אתר ה-web של mottyN מציג יתר אירועים כי Firestore שלו לא מוחק סגורים — באג שאנו נמנעים ממנו.)
- **מעבר מסננים** — מקור הנתונים מתצורה, לעקיפת סינון תוכן (NetFree) במכשירים מנוהלים.
- **עמידות** — אין תלות ב-Google Play Services; המפה נוצרת ב-runtime ב-try/catch (מכשיר ללא
  WebView לא קורס).

## יכולות

- 🔴 חלון התראה מלא שמצלצל מעל המסך הנעול + רטט + מפת OSM
- 🧭 ניווט בלחיצה ל-**Waze** (נפילה ל-geo: chooser)
- 🚨 כפתור "דיווח על מעצר" — חיוג למוקד צבע שחור עם שלוחה אוטומטית
- 📜 היסטוריה מקומית (שומרת עריכות) + היסטוריית שרת, עם פרטי אירוע מלאים ומפה בלחיצה
